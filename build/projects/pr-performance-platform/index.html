PR Performance Platform

Senior Software Engineer · 2024 · 1 min read

Reduced frontend regressions by 40% and improved developer confidence by providing automated performance feedback on every Pull Request.

Overview

At Atlassian, we needed a way to catch frontend performance regressions before they reached production. I led the research and implementation of a platform that captures performance metrics at the PR stage.

Problem

Performance regressions were often caught too late, leading to expensive fixes and degraded user experience in Jira and Confluence.

Constraints

  • Must support multiple monorepos
  • Low overhead on CI/CD pipelines
  • Accurate and reproducible metrics

Approach

We built a specialized testing runner that executes in a controlled environment, comparing PR changes against the main branch using Playwright and custom tracing logic.

Key Decisions

Use Synthetic Lab Testing over RUM for PRs

Reasoning:

Synthetic testing provides the stable, reproducible environment needed to compare specific code changes without noisy production variables.

Implement an 'Impact Score' instead of raw milliseconds

Reasoning:

Raw time varies; a relative impact score helps developers understand the significance of a change more intuitively.

Tech Stack

  • TypeScript
  • Node.js
  • Playwright
  • AWS
  • Go

Result & Impact

  • 40%
    Regression Reduction
  • < 10 mins
    Feedback Time

Shifted the performance culture from reactive to proactive across the entire engineering organization.

Learnings

  • Developer experience (DX) is as important as the tool's accuracy for adoption.
  • Automated guardrails are more effective than manual performance reviews.
